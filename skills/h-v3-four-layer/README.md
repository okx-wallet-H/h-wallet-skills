# H V3 四层 AI 合约策略信号系统

## 架构

```
┌─────────────────────────────────────────────────────────────┐
│                    L4: Bot 推送层                             │
│           h_v3_bot_v2.py (Telegram Bot)                      │
│     定时推送 / 命令交互 / AI分析 / 格式化输出                  │
├─────────────────────────────────────────────────────────────┤
│                    L3: 回测验证层                             │
│           h_v3_backtest.py (VectorBT PRO)                    │
│     胜率 / 盈亏比 / 最大回撤 / Sharpe / 绩效摘要             │
├─────────────────────────────────────────────────────────────┤
│                    L2: 策略引擎                               │
│           h_v3_strategy.py (8因子评分)                        │
│     趋势/动量/MACD/布林/资金流/市场结构/聪明钱/多TF           │
├─────────────────────────────────────────────────────────────┤
│                    L1: 数据接口层（可插拔）                    │
│           h_v3_data_api.py (OKX CLI适配器)                   │
│     70+技术指标 / 资金费率 / 持仓量 / 聪明钱 / 多TF          │
└─────────────────────────────────────────────────────────────┘
```

## 核心设计原则

1. **数据源可插拔**：引擎只认标准格式数据，不关心数据从哪来
2. **解耦不重构**：数据层和策略层完全分离，换数据源不影响策略
3. **秒回架构**：后台每5分钟刷新缓存，用户请求直接读缓存
4. **容错降级**：任何单个数据源失败不阻塞，保留上次有效数据

## 文件说明

| 文件 | 层级 | 说明 |
|------|------|------|
| h_v3_data_api.py | L1 数据接口层 | OKX CLI 全能力封装，并行调用，后台缓存 |
| h_v3_strategy.py | L2 策略引擎 | 8因子多维度评分，±9.0分制，ATR止损止盈 |
| h_v3_backtest.py | L3 回测验证 | VBT PRO + 快速回测，绩效数据附带信号 |
| h_v3_bot_v2.py | L4 Bot推送 | Telegram Bot，定时推送，命令交互 |

| 因子 | 权重 | 数据来源 |
|------|------|----------|

| 聪明钱 (Top交易员共识) | 1.3 | OKX CLI smartmoney |
| 多时间框架 (1H/4H/1D共振) | 1.0 | OKX CLI multi-TF |

## Bot 命令

- `/start` - 系统介绍
- `/signal [币种]` - 获取信号（默认BTC）
- `/all` - 全部币种信号
- `/status` - 系统状态
- `/backtest [币种]` - 运行回测
